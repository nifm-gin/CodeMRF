Les nouvelles matrices de rotation tenant compte du df sont contenues dans les fichiers  dico_spinsx10_df_full_..._corr.mat, la version sans le "_corr" contient les matrices avant la correction d'une erreur: le déphasage dû au df pendant le pulse était compté dans les matrices de rotation et recompté plus tard dans le code de simulation du dictionnaire.
Le code de calcul du dictionnaire ne permet pas pour le moment de tenir compte de ces énormes matrices, d'autant plus qu'elles sont contenues dans plusieurs fichiers distincts et qu'on ne peut en charger qu'un à la fois sous peine de saturer la mémoire vive. Il faudrait donc, au début du calcul, charger le premier fichier, extraire les matrices qui seront utiles (df, FA*B1rel, v utiles pour le dico), supprimer le reste, charger le 2e fichier, extraire les matrices utiles, supprimer le reste, etc. De plus il faudra modifier le code pour qu'il tienne compte de la dimension supplémentaire de la grande matrice Rot (qui s'appelle ici Rot_corr d'ailleurs).
